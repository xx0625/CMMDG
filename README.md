# CMMDG: Cross-database Multi-expert Domain Generalization for Cognitive Workload Assessment

基于跨数据库多专家域泛化的认知工作负荷评估框架，支持 EEG 信号的多数据库交叉验证（LODO: Leave-One-Database-Out）。
A cross-database multi-expert domain generalization framework for cognitive workload assessment, supporting multi-database cross-validation (LODO: Leave-One-Database-Out) on EEG signals.

## 数据准备 / Data Preparation

### 1. 获取原始数据 / Obtain Raw Data

项目使用以下四个公开 EEG 数据集：
The project uses the following four public EEG datasets:

| 数据集 | 任务 | 导联数 | 格式 | 获取方式 |
|--------|------|--------|------|----------|
| Dataset | Task | Channels | Format | Source |
| BOOLS-EX1 (nBack) | N-back 工作记忆 / N-back working memory | 14 共享导联 | .edf | 已包含在 `data/raw_data/BOOLS-EX1/` / Included in `data/raw_data/BOOLS-EX1/` |
| MATB (COG-BCI) | 多属性任务电池 / Multi-attribute task battery | 56 / 14 共享 | .set (EEGLAB) | 需从官方源申请 / Request from official source |
| MOBA (EEGMOBA) | 手机MOBA游戏 / Mobile MOBA game | 56 / 14 共享 | .vhdr (BrainVision) | 需从官方源申请 / Request from official source |
| STEW | 同时多任务 / Concurrent multitasking | 14 共享导联 | .txt | 需从官方源申请 / Request from official source |

### 2. 运行预处理 / Run Preprocessing

```bash
cd code/preprocessing
python preprocess_all.py                    # 处理所有数据 / Process all data
python preprocess_all.py --dataset matb     # 仅处理 MATB / Process MATB only
python preprocess_all.py --channels 56      # 仅处理 56 导联 / Process 56 channels only
```

预处理输出将保存在 `data/process_data/` 目录下。
Preprocessing outputs will be saved in the `data/process_data/` directory.

## 使用方法 / Usage

### 训练 CMMDG 主模型 / Train CMMDG Main Model

```bash
cd code/ours
python train.py
```

支持三种模式（在 `train.py` 中修改 `MODE` 变量）：
Three modes are supported (modify the `MODE` variable in `train.py`):

- `LODO`：4 折交叉验证（4 个数据库，14 导联）/ 4-fold cross-validation (4 databases, 14 channels)
- `LODO-SL`：2 折交叉验证（nback + stew，14 导联）/ 2-fold cross-validation (nback + stew, 14 channels)
- `LODO-DL`：2 折交叉验证（matb56 + mg56，56 导联）/ 2-fold cross-validation (matb56 + mg56, 56 channels)

### 运行对比模型 / Run Comparison Models

每个对比模型有独立的验证脚本，例如：
Each comparison model has its own validation script, e.g.:

```bash
cd code/comparemode/train/两个14通道/普通模型
python 14_dcnn_val.py
python 14_eegnet_val.py
# ... 其他模型 / ... other models
```

### 查看结果汇总 / View Results Summary

```bash

cd code/comparemode/train/两个14通道/普通模型
```

## 训练流程说明 / Training Pipeline Description

CMMDG 模型的训练包含四个阶段：
The CMMDG model training consists of four stages:

1. **Stage 1 - 预训练**：使用全部训练数据进行多专家域泛化预训练
   / **Stage 1 - Pre-training**: Multi-expert domain generalization pre-training using all training data
2. **Stage 2 - 样本筛选**：基于置信度-相似度-鲁棒性的综合打分，剔除低质量样本
   / **Stage 2 - Sample Selection**: Comprehensive scoring based on confidence-similarity-robustness, removing low-quality samples
3. **Stage 3 - 重训**：在净化后的数据上重新训练模型
   / **Stage 3 - Retraining**: Retrain the model on the purged data
4. **Stage 4 - 测试**：在目标域上评估模型性能
   / **Stage 4 - Testing**: Evaluate model performance on the target domain

## 关键参数说明 / Key Parameter Description

主要训练参数在 `code/ours/train.py` 的 `CONFIGS` 字典中配置：
Main training parameters are configured in the `CONFIGS` dictionary in `code/ours/train.py`:

- `ALPHA`, `BETA`, `GAMMA`：样本打分权重 / Sample scoring weights
- `PRUNING_RATIO`：样本剔除比例 / Sample pruning ratio
- `PPT_W1`, `PPT_W2`：因果保持损失权重 / Causality-preserving loss weights
- `W_CONS`, `W_EXP`, `W_DOM`：一致性损失、专家损失、域判别损失权重 / Consistency loss, expert loss, domain discrimination loss weights

## 注意事项 / Notes

1. **数据路径**：预处理后的数据默认保存在项目根目录的 `data/process_data/` 下，训练脚本会自动从该路径读取
   / **Data Path**: Preprocessed data is saved in `data/process_data/` by default, training scripts automatically read from this path
2. **随机种子**：所有实验使用固定的随机种子（`seed=42`），确保结果可复现
   / **Random Seed**: All experiments use a fixed random seed (`seed=42`) to ensure reproducibility
3. **GPU 要求**：推荐使用 CUDA 兼容的 GPU 进行训练（显存 >= 8GB）
   / **GPU Requirements**: CUDA-compatible GPU is recommended for training (memory >= 8GB)
4. **检查点**：模型检查点默认保存到 `code/ours/train_output/` 目录
   / **Checkpoints**: Model checkpoints are saved to `code/ours/train_output/` by default
5. **对比模型**：部分对比模型（如 SCVCNet、FBNet）需要额外的滤波预处理数据，请参考对应脚本的说明
   / **Comparison Models**: Some comparison models (e.g., SCVCNet, FBNet) require additional filtered preprocessing data, please refer to the corresponding script documentation
